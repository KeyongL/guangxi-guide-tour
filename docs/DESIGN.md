> 生成于 2026-07-03 /design 会话。风格三问已由用户确认：

> 域名未定，正式名称由站长购买域名时定夺——SPEC 已提醒品牌不要绑死"Beihai"）。

> **⚠️ 方向变更 2026-07-03（本文件唯一现行方向，覆盖上方"旅行编辑风"）：**
> 用户经三轮反馈定稿新方向 = **极简 / 北欧风 / 中英对照 / 不说废话**。
> 与原"旅行编辑风"的差异（新方向为准）：
> - **无衬线取代衬线**：标题与正文统一系统无衬线栈，去掉 Georgia 衬线；标题 `font-weight:650 letter-spacing:-.01em`。
> - **直角取代圆角**：卡片/图片/按钮一律 `border-radius:0`（徽章可保留描边，不用胶囊填充）。
> - **细线取代色块**：1px 描边 + 分隔线承担结构；去渐变、去阴影、去 emoji 装饰。
> - **扁平照片占位**：`background:var(--primary-soft)` + 1px 描边 + 灰字图注，不用灰绿渐变。
> - **留白加倍、文案砍半**：节间距更大；一屏只给必要信息，其余折叠（city-card 用原生 `<details>`，零 JS）。
> - **中英对照（2026-07-03 收窄，用户认可）**：双语只留在"游客会指给本地人看的名词"——**地名 / 菜名 / 地址**（如 Weizhou 涠洲岛）。**UI 标签、按钮、导航、正文一律纯英文**——目标读者英语母语读不懂中文，UI 双语是噪音、服务的是管理者不是游客；中文归到管理/CMS 层。此条覆盖上文"UI 标签双语"的旧记法。
> 色彩令牌不变（仍用下方深浅两套 hex）；琥珀 `--accent` 收窄为仅签证警告 + 唯一主 CTA。
>
> **页型区分（2026-07-03 用户认可）：**
> - **着陆/枢纽页 = 地图为主角**（样张 `07-map-first.html`）：答案式大标题 → 大地图英雄第一屏 → 点城市弹极简名片 → "Start here" 几个链接。文字砍到最少，签证分区等复杂度**点击展开**（渐进披露），不进英雄图。
> - **深度攻略页 = 答案式开头 + 正文折叠**（样张 `03-article.html` 需按此再调）：首屏先给"你这护照行不行"的结论，再折叠展开正文；避免 14min+目录+铺垫的教科书姿态。
>
> **反劝退设计原则（来自 2026-07-03 失败分析）：**
> 1. 结论前置——首屏给定心丸，不给研究任务。
> 2. 地图是钩子不是附件——用差异化资产(路线地图)开场，不用文字开场。
> 3. 信任信号克制——别把 14min/博士/官方框/警告框堆成官僚感/法务页。
> 4. 复杂度渐进披露——签证分区/警告点击再现。
> 5. 手机优先——桌面式密度到 375px 活不下来。
>

## 设计原则

1. **照片即内容**：大幅真实照片主导版面，UI 退后。原型中用灰阶渐变占位块+图注标明"这里放什么照片"。禁止插画风、禁止 AI 生成图充当实拍。
2. **编辑部感，不是模板感**：衬线大标题 + 正文舒适行距 + 充足留白；卡片克制圆角（8px），无重阴影，无渐变按钮。参照独立旅行杂志而非 SaaS 落地页。
3. **信任前置**：每篇文章显示作者、实地核实日期（"Last verified on the ground: …"）、官方来源链接。About 页是转化核心。
4. **签证分区防错是系统职责**：崇左/百色内容必须由模板组件自动挂"签证适用性提醒框"，不靠作者记忆（对应验收标准 7）。
5. **移动优先**：375px 无横向滚动；性能预算 = 系统字体 + 无 JS 框架 + 图片懒加载（对应验收 3、6）。

## 字体（零外部请求：系统字体栈）

- 标题/Display（衬线，编辑风核心）：`Georgia, 'Iowan Old Style', 'Times New Roman', serif`
- 文章正文（衬线，长文阅读）：同上，`1.0625rem / 1.75`
- UI 元素（导航、按钮、表单、徽章，无衬线）：`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- 字号阶梯：Display `clamp(2rem, 5vw, 2.75rem)`；H2 `1.5rem`；H3 `1.1875rem`；UI `0.9375rem`；图注/小字 `0.8125rem`
- 正式开发若需品牌感可换自托管 Source Serif 4（Cloudflare Pages 同域加载），原型阶段不引入。

## 间距与布局

- 基准 8px；节间距移动端 `48px`、桌面 `80px`
- 版心：整站 `1100px`；文章正文 `700px`（每行 ~65 字符）
- 卡片圆角 `8px`，描边 `1px solid var(--border)`，无阴影（hover 时 `border-color: var(--primary)`）
- 断点：单断点 `768px`（移动优先，桌面增强）
