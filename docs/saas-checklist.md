# SaaS 市场化产品完整清单
# 供 /intake、/init-harness、/verify 引用
# 用户选择"个人工具"时忽略此文件；选择"市场化产品"时逐项核对

## 必须有的层（缺一个上不了线）

### 1. 用户认证
- [ ] 邮箱注册 + 密码
- [ ] 邮箱验证（注册后发验证邮件）
- [ ] 忘记密码 / 重置密码
- [ ] OAuth 登录（Google / GitHub / 微信——按目标用户定）
- [ ] 登录状态保持 + 安全退出
- [ ] 基础角色（普通用户 / 管理员）

### 2. 计费 & 订阅
- [ ] 海外：Stripe 订阅计划（月付 + 年付）+ Webhook 处理
- [ ] 国内：微信支付 + 支付宝（需申请商户号）
- [ ] 免费试用期 / Freemium 计划
- [ ] 订阅到期处理（降级而不是直接锁账户）
- [ ] 退款处理流程

### 3. 数据库
- [ ] 用户表 + 业务核心表
- [ ] 数据库迁移脚本（schema 变更可追踪）
- [ ] 多租户数据隔离（用户只能看自己的数据）
- [ ] 定期备份策略

### 4. 云端部署（双栈）
- [ ] 海外：Vercel / Railway（自动 CI/CD，GitHub push 自动部署）
- [ ] 国内：阿里云 / 腾讯云（需 ICP 备案，约 20 个工作日）
- [ ] 环境分离：开发 / 测试 / 生产 三套环境
- [ ] 域名：海外域名 + 国内域名（.cn 或带备案的）
- [ ] HTTPS 自动续期

### 5. 邮件系统
- [ ] 发信服务：Resend（海外）/ 阿里云邮件（国内）
- [ ] 注册欢迎邮件
- [ ] 邮箱验证邮件
- [ ] 密码重置邮件
- [ ] 订阅到期提醒邮件

### 6. 安全
- [ ] API 限速（防刷）
- [ ] 敏感操作二次确认
- [ ] 输入校验（XSS / SQL 注入防护）
- [ ] Secrets 管理（API Key 不进 git，用环境变量）
- [ ] 依赖库安全扫描

### 7. 监控 & 运营
- [ ] 报错追踪：Sentry（能收到真实用户的报错）
- [ ] 用户行为分析：PostHog / GA4
- [ ] API 成本监控 + 告警（AI 调用花多少钱）
- [ ] 服务可用性监控（宕机自动通知你）

## 上线前必须有的页面

### 产品页面
- [ ] 落地页（首页，有产品介绍、定价、FAQ）
- [ ] 定价页
- [ ] 登录 / 注册页
- [ ] 用户仪表盘（核心功能入口）
- [ ] 账户设置页（改密码、管理订阅）

### 法律页面（缺了用户不信任，App Store 不过审）
- [ ] 隐私政策（Privacy Policy）
- [ ] 服务条款（Terms of Service）
- [ ] Cookie 政策（如果用了 Cookie）
- [ ] 国内需要：《个人信息保护政策》（符合《个保法》）

### 管理后台（你自己用，不对外）
- [ ] 用户列表 + 搜索
- [ ] 手动管理订阅（退款、升降级）
- [ ] 系统公告推送
- [ ] API 用量统计

## 国内特殊要求

- [ ] ICP 备案（在工信部，约 20 个工作日，服务器必须在国内）
- [ ] 公安网安备案（ICP 通过后 30 天内做）
- [ ] 如涉及用户生成内容：内容安全合规（接入阿里云内容安全 API）
- [ ] 如有 App：工信部 App 备案

## 好用户体验必须有的（影响留存）

- [ ] 用户引导流程（新用户注册后的第一步引导）
- [ ] 空状态设计（没有数据时展示什么）
- [ ] 加载状态（等待时用户看到什么）
- [ ] 操作成功 / 失败反馈（Toast 提示）
- [ ] 移动端响应式
