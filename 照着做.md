──────────────────────────────────────────────
  第二部分：开始做项目
──────────────────────────────────────────────

每个命令 = 一个新对话。做完关掉，再开新的。
每个命令结束时 AI 都会告诉你下一步（📍 标记），照做就行。

【第 0 步】/start-here — 确认目录
   AI 报告它在哪个文件夹。你核对，回"确认"。

【第 1 步】/intake — 问清需求
   AI 一题一题问你。市场化产品会自动核对 SaaS 清单。
   确认需求文档后，AI 告诉你下一步（/research 或 /init-harness）。

【第 2 步】/research — 搜索技术栈 + 找开源底子（小工具可跳过）
   AI 搜索最新实践出对比表，同时推荐可以当底子的开源项目。
   你选方案 → 如果有开源底子,AI 告诉你怎么下载（一行命令）
   → 下载后打 /adopt 接管（不打 /init-harness，因为已有代码）。

【第 3 步】/design — 设计 + 看原型画面（无 UI 可跳过）
   AI 问你风格偏好 → 直接生成 HTML 原型。
   你在浏览器打开看。满意说"可以"，不满意提修改意见。

【第 4 步】/prototype — 技术验证（小项目可跳过）
   AI 只把核心路径跑通（如：能连数据库、能调 AI、能支付）。
   不是完整产品，是"验证路走得通"。

【第 5 步】/init-harness — 建功能清单和项目骨架
   AI 自动检测有没有 Boilerplate，有就在上面建，没有就从头搭。
   清单建好后所有功能标 ✗，之后就是把 ✗ 一个个变 ✓ 的过程。

【第 6 步】做功能（两种方式选一个）

  方式 A：手动（每次开新对话，打 /work）
    AI 先读进度，挑一个 ✗ 做完变 ✓，停下。
    每 3-5 个穿插一次 /verify。适合你想全程盯着看的情况。

  方式 B：自动循环（去吃饭/睡觉时用）
    在终端运行：python pipeline.py
    它会自动链式执行 /work → /verify → /work → ...
    每个任务在独立子进程里（全新上下文，不会爆炸漂移）
    进度靠 feature_list.json 保存，Ctrl+C 暂停后可续跑。
    → 最后一个做完时自动运行最终质检

【第 7 步】/verify — 最终验收
   AI 挑剔地重测全部功能。全过 → AI 告诉你下一步。

【第 8 步】/launch — 发布前核查（市场化产品）
   逐项核对：支付环境切换、法律页面、域名 HTTPS、备案等。
   全过 → 可以上线。

其他命令（需要时才用）：
  · 给项目做安全+健康体检 → /audit（每月一次，上线前必做）
  · 给项目大扫除（清死代码、归档废弃项）→ /cleanup（每月一次）
  · 文件乱了/复用代码后重新规整 → /reorganize（按标准搬家归位）
  · agent 上生产前加硬护栏（格式校验/危险操作确认）→ /harden
  · 不想手动一次次切 session → /pipeline（看怎么启动自动循环）
    或直接在终端运行 python pipeline.py，work+verify 自动链式
  · 离开电脑让 AI 连做多个功能（吃饭/睡觉）→ /auto
    （前提：功能有自动化测试 + 设 token 上限）
  · 想改需求（加减功能） → /change
  · 方向错了（技术栈/定位/整体设计不对） → /pivot
  · AI 卡住了 → /checkpoint（存档退出，开新对话继续）
  · 已有成品 → /adopt（代替第 1-5 步）
  · 不确定在哪 → /status
