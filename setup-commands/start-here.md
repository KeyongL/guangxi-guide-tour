# /start-here — 开工前第一件事：锁定工作目录

你的唯一任务是和用户确认工作目录，然后停下等待。**本命令不创建任何文件、不写代码。**

## 步骤

1. 运行 `pwd`，把完整路径清清楚楚地报告给用户，例如：
   "我现在的工作目录是：/Users/mac/Desktop/AI workflow/文献采集智能体"

2. 运行 `ls -la`，把当前目录下已有的文件/文件夹列给用户看。

3. 明确询问用户："以上就是你想要工作的目录吗？接下来所有文件都会在这里创建。
   确认请回复'确认'，需要换目录请把正确的完整路径发给我。"

4. 停下等待用户回应。**在用户确认之前，不要进行任何后续操作。**

5. 用户确认后，告诉用户："目录已锁定。接下来：
   - 如果是全新项目，开新对话打 /intake
   - 如果是已有成品代码，开新对话打 /adopt
   我后续所有工作都只在这个目录进行，不会擅自切换。"

## 铁律（写给所有命令的总纲）
- **绝对禁止 cd 到用户未指定的目录**。
- **绝对禁止因为别处存在一个看起来更像"项目"的同名文件夹，就切换过去工作**。
  哪怕当前目录是空的，也要先问用户，而不是自己去别处找。
- 当前目录与用户期望不符时，停下来问，不要脑补、不要自作主张。
