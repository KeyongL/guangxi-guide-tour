你的唯一任务：读取项目文件，用最简单的语言告诉用户现在在哪、该干嘛。

## 读取顺序

按以下顺序检查文件是否存在，判断当前阶段：

1. 有没有 `docs/SPEC.md`？
2. 有没有 `docs/DESIGN.md`（如果 SPEC 里有 UI）？
3. 有没有 `mockups/` 目录且里面有 HTML 文件？
4. 有没有 `prototype/` 目录且原型验证通过的记录？
5. 有没有 `feature_list.json`？
6. 如果有 feature_list.json：`passes: true` 有几个 / 总共几个？
7. `claude-progress.txt` 最后 10 行写的是什么？
8. 有没有 `LAUNCH_COMPLETE` 标记文件？

## 输出格式（严格按这个格式，不要多写）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 项目状态 · [项目名或目录名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📍 当前阶段：[下面选一个]
   · 【待启动】还没有需求文档
   · 【需求阶段】SPEC 已有，还没确认技术栈
   · 【设计阶段】技术栈已定，还没看到界面原型
   · 【技术验证】原型已确认，还没跑通技术路线
   · 【开发中】正在逐个做功能
   · 【收尾验收】所有功能已完成，等待最终质检
   · 【✅ 已完成】质检通过，等待上线

📈 开发进度：X / Y 个功能完成（Z%）
   [用一行简单的进度条，例如：▓▓▓▓▓▓░░░░ 60%]

🕐 上次做的事：[从 claude-progress.txt 提取一句话摘要]

⚡ 下一步（直接照做）：
   [只写一句话，例如：]
   · "开新对话，打 /work，做第 8 个功能"
   · "开新对话，打 /verify，已经 5 个功能没质检了"
   · "开新对话，打 /research，技术栈还没确认"
   · "🎉 所有功能完成！开新对话，打 /verify 做最终验收"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 判断"该打 /verify 了"的条件

满足以下任一条件，下一步提示 /verify 而不是 /work：
- 上次 /verify 之后新完成了 3 个以上功能
- claude-progress.txt 里显示连续 5 次 /work 没有做过 /verify
- feature_list.json 里 passes:true 的数量刚好跨过了 25% / 50% / 75% / 100% 的整数关口

## 判断"项目完成"的条件

同时满足以下三条：
1. feature_list.json 里所有条目 passes 都是 true
2. claude-progress.txt 里有"最终 /verify 通过"的记录
3. 没有任何 passes:false 的条目

此时输出：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎉 项目开发完成！

所有 Y 个功能已通过验证。

下一步是上线：
  开新对话，打 /launch 做发布前核查
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 铁律
- 输出必须在 20 行以内，不写分析，不写建议，只写事实和下一步。
- 下一步只写一个，不写"你可以做 A 或者 B"，直接告诉用户做哪个。
- 如果文件都不存在，说明是全新目录，输出：
  "这是新项目目录。第一步：打 /start-here 确认工作目录。"
